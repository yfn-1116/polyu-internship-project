# PRD：SMPL 人体建模服务

> 状态：Draft
> 最后更新：2026-06-05
> 目标读者：后续开发者、实习项目接手者、导师评审者

## 1) 背景、目标与成功标准

### 1.1 背景

本项目要开发一套独立的 SMPL 人体建模服务，作为大型医疗体态系统的子组件。现阶段使用模拟数据和公开人体数据样本开发，后续接入医院 3D 扫描仪真实数据，并通过稳定接口对接主业务系统。

### 1.2 总体目标

- 可插拔：输入数据源、模型 backend、输出方式都能替换。
- 可扩展：从 mock 样本扩展到公开数据集，再扩展到医院扫描仪。
- 可复现：代码、设计文档、运行命令、开发日志完整保留。
- 可交接：新开发者能通过 `99-project-planning.md` 和 `97-journal.md` 继续推进。

### 1.3 非目标

- 第一周不从零训练 SMPL。
- 第一周不做完整医疗业务系统、病例管理和用户权限。
- 第一周不做商业级高真实感渲染、皮肤/衣物/纹理重建。
- 第一周不处理真实患者隐私数据。

### 1.4 成功标准

第一周最小成功标准：

- 能运行一次“输入样本 -> 模型 backend -> 输出 manifest”的完整流程。
- backend 可替换：真实 SMPL/SMPL-X 可用时接真实模型，不可用时 mock backend 不破坏主流程。
- 输出包含可被后续系统读取的结构化 manifest。
- 提供最小前端 3D Viewer，能加载生成的模型或样例模型并完成旋转、缩放、查看。
- 文档能说明目标、架构、运行方式、风险和下一步。

## 2) 需求边界与待确认项

### 2.1 输入数据

- 当前：mock/synthetic 样本。
- 优先：THuman2.0 或类似 3D human scan/mesh 数据。
- MoCap：AMASS ElenaKyriakou 动作捕捉数据（SMPL+H 格式，13 种情绪，驱动 SMPL-X 生成动画序列）。
- 后续：医院 3D 扫描仪数据。

### 2.2 输出结果

最小输出：

- `manifest.json`
- mesh 或 params 文件路径
- task_id、model_type、status、errors

后续输出：

- 关节位置
- 体态测量指标
- 可视化预览
- 医疗系统报告字段
- 动画序列（MoCap 驱动的多帧 mesh 数据）

### 2.3 前端展示

第一周需要一个最小可展示前端，用于证明 pipeline 结果可以被用户直观看到。

最小范围：

- 加载后端输出或样例 mesh。
- 支持旋转、缩放、平移。
- 展示任务状态和 manifest 核心字段。
- 不追求商业级真实皮肤、衣物、纹理和光照。

后续产品化方向：

- 医疗体态测量 overlay。
- 正面、侧面、背面对比。
- 历史模型对比。
- 更真实的材质、光照、纹理和报告展示。
- MoCap 动画播放：加载后端输出的动画序列，支持情绪切换和播放控制。

### 2.4 待确认

- 第一周验收是否必须使用真实公开数据集。
- 第一周前端是否必须通过 HTTP API 读取结果，还是可先加载本地样例。
- 本机是否已有 SMPL/SMPL-X 模型文件下载权限。
- 后续医院扫描仪输出格式。

## 3) 全局原则

- GP-1 服务边界清晰：本服务只做人体现建模，不做医疗业务系统。
- GP-2 Contract-first：输入、输出、manifest 字段先稳定，再扩展内部实现。
- GP-3 可替换 backend：mock、SMPL、SMPL-X 通过同一模型接口接入。
- GP-4 数据不入库原则：公开数据集和模型权重不提交到 Git。
- GP-5 文档同步：代码、接口、运行方式变化必须同步更新对应文档。

## 4) 关键用户流程

### UJ-1 开发者本地验证

1. 安装依赖。
2. 准备 mock 或样本输入。
3. 运行 CLI。
4. 查看输出 manifest 和结果文件。
5. 根据日志定位失败原因。

### UJ-2 后续接入公开数据集

1. 下载或申请公开人体数据集。
2. 写入 `DatasetInputAdapter`。
3. 转换为标准 `BodyInput`。
4. 复用同一模型 backend 和输出流程。

### UJ-3 后续接入医院扫描仪

1. 确认扫描仪输出格式。
2. 新增 `ScannerInputAdapter`。
3. 转换为标准 `BodyInput`。
4. 保持主 pipeline 和输出 contract 不变。

## 5) MVP 定义

MVP 是一个可复现、可交接的 SMPL 建模服务最小闭环：

```text
sample input -> InputAdapter -> ModelBackend -> OutputWriter -> manifest -> 3D Viewer
```

验收以 `documents/99-project-planning.md` 中执行队列 DoD 为准。
