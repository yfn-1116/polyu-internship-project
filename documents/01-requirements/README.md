# Requirements 目录说明

本目录记录项目“要做什么、不做什么、做到什么算完成”。主入口是 `01-prd.md`。

## 记录目标

- 让接手者快速理解项目目标、边界、用户、MVP 和验收标准。
- 将功能需求、非功能需求和待确认问题集中在 PRD 中，避免分散。
- 需求文档只写产品和工程目标，不写模块内部设计。

## 当前文档

- `01-prd.md`：项目 PRD，包含背景、目标、非目标、原则、MVP、验收。

## 更新规则

- 目标或验收变化：更新 `01-prd.md`。
- 设计方案变化：更新 `documents/02-design/**`。
- 实现进展：更新 `documents/97-journal.md` 和 `documents/99-project-planning.md`。
