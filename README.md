# PolyU Internship Project

This repository stores project documentation, planning records, architecture notes, and handoff material for the Hong Kong Polytechnic University internship project.

The current repository is intentionally a documentation shell. Requirements, design, technology choices, implementation details, and operational runbooks will be refined as the project scope becomes clearer.

## Repository Structure

```text
.
├── documents/
│   ├── 01-requirements/
│   ├── 02-design/
│   ├── 03-faq/
│   ├── 04-challenges/
│   ├── 98-runbook/
│   └── 99-knowledge-base/
├── investment/
├── 99-project-planning.md
└── README.md
```

## Documentation Goals

- Keep requirements, design, and development history understandable for future maintainers.
- Record why decisions were made, not only what was changed.
- Separate frontend, backend, data, and deployment concerns when the architecture becomes clearer.
- Make project handoff possible even if the original developer stops working on the project.

## Working Rules

- Write documentation in English.
- Keep development notes in `99-project-planning.md`.
- Put reusable explanations, references, and decisions into the relevant `documents/` folder.
- Commit changes regularly with clear Git messages.
